# API Gaps

| Provider | Gap | Status | Exact verification step |
| --- | --- | --- | --- |
| Toss | Auth token issue/refresh flow | verified-readonly-implemented | Official OpenAPI JSON confirms `POST /oauth2/token`, OAuth2 Client Credentials Grant, form fields `grant_type`, `client_id`, `client_secret`, and no refresh token |
| Toss | account/position query schema | verified-readonly-implemented | Official OpenAPI JSON confirms `GET /api/v1/accounts`, `GET /api/v1/holdings`, and `X-Tossinvest-Account=accountSeq` |
| Toss | cash buying power query | verified-readonly-implemented | Official OpenAPI JSON confirms `GET /api/v1/buying-power?currency=KRW` with `cashBuyingPower` decimal string |
| Toss | quote/current price query | verified-readonly-implemented | Official OpenAPI JSON confirms `GET /api/v1/prices?symbols=...` result fields `symbol`, `timestamp`, `lastPrice`, `currency` |
| Toss | KR market calendar | verified-readonly-implemented | Official OpenAPI JSON confirms `GET /api/v1/market-calendar/KR` with `today.integrated.regularMarket.startTime`, optional `singlePriceAuctionStartTime`, and `endTime` |
| Toss | candle/history query | verified-readonly-implemented | Official OpenAPI JSON confirms `GET /api/v1/candles` params `symbol`, `interval`, `count`, `before`, `adjusted` and `CandlePageResponse` |
| Toss | order status read/reconciliation | verified-readonly-implemented | Official OpenAPI JSON confirms `GET /api/v1/orders` and `GET /api/v1/orders/{orderId}` as read-only order history/status APIs; worker maps official status enum values to local order state and fails unknown codes to manual review |
| Toss | guarded order create execution | verified-limit-implemented | Official OpenAPI JSON confirms `POST /api/v1/orders`, `OrderCreateRequest`, `clientOrderId` max length 36, `LIMIT` quantity+price payload, and account header. Implementation is limited to worker-only KRX quantity-based `LIMIT` orders after final `RiskService` pass and fresh manual live approval controls |
| Toss | live account state sync for scheduled cycle | partial-system-only-accepted-fail-closed | Scheduled live cycles use official Toss `buying-power` and `holdings` for cash/equity/daily PnL. Broker-wide or externally placed daily order history remains unverified because official `PaginatedOrderResponse` currently documents `status=CLOSED` as `400 closed-not-supported`; the worker can enforce only system-created live order count from local `orders` rows for the current KST trading day. Production live operation must therefore be explicitly limited to system-originated orders by setting `LIVE_SYSTEM_ORDER_COUNT_SCOPE_ACCEPTED=true`; without runtime acceptance, the worker records `live_external_order_history_scope_not_accepted` and blocks with `daily_order_count_unverified` before any broker call. Release readiness must also retain `system_order_scope_evidence.json` proving the exact scope, Toss limitation, deployment environment, operator, runtime env confirmation, HTTPS evidence URI, and SHA-256 hash; missing or weak/non-HTTPS retained evidence blocks bundle creation. The final bundle rejects environment mixing: `staging` requires `deployment_environment=staging`, and `production-readiness` requires `deployment_environment=production` |
| Toss | guarded order cancel execution | verified-limit-implemented | Official OpenAPI JSON confirms `POST /api/v1/orders/{orderId}/cancel`, empty optional JSON body, and `OrderOperationResponse.orderId` as a new cancel operation id. Implementation is limited to worker-only manual cancellation by local `orders.id` for `sent`/`partial_filled` live orders; the worker then confirms the original order via `GET /api/v1/orders/{orderId}` and marks local `canceled` only after official `CANCELED`. Timeout, unknown provider result, or non-`CANCELED` confirmation sets `unknown_requires_manual_check` |
| Toss | order modify execution | verified-not-implemented | Official OpenAPI JSON confirms `POST /api/v1/orders/{orderId}/modify`; do not implement until price/quantity policy, rollback, and operator playbooks are complete |
| KRX | market calendar endpoint | verified-not-implemented-replaced-by-toss | KRX Open API market-calendar product is not implemented or used in the live/non-mock worker path; Korean market-open gating is supplied by the already verified Toss `GET /api/v1/market-calendar/KR` contract, and `HealthService` now records live market-data health as `toss_market_data` instead of mislabeling it as `krx` |
| KRX | listing/sector statistics | verified-not-live-critical | KRX listing/sector APIs remain outside the live trading safety path; the current worker does not depend on KRX listing/sector data for order creation, and any direct `KrxClient` usage remains fail-closed until exact product approval, endpoint, params, and schema are verified |
| OpenDART | corp code mapping | verified-readonly-implemented | Official OpenDART guide confirms `GET /api/corpCode.xml?crtfc_key=...` returns a ZIP containing `CORPCODE.xml` with `corp_code`, `corp_name`, `stock_code`, and `modify_date`; worker parses the official XML shape, maps Korean stock code to `corp_code`, rejects oversized ZIP/XML and DTD/entity declarations before XML parsing, and fails malformed ZIP/XML closed |
| OpenDART | account name mapping | verified-partial-fundamentals-implemented | Official OpenDART financial statement guide confirms `GET /api/fnlttSinglAcntAll.json` with `corp_code`, `bsns_year`, `reprt_code`, and `fs_div`, returning rows with `account_nm` and current amount fields; worker maps Korean account names for revenue, operating income, net income, assets, liabilities, and equity to compute `operating_margin`, `roe`, and `debt_ratio`. `per`/`pbr` remain `None` because they require market price/capitalization data outside OpenDART financial statements |
| Naver | rate limits and error schema | verified-readonly-implemented | Official Naver Search News API docs confirm `GET https://openapi.naver.com/v1/search/news.json`, required `X-Naver-Client-Id` and `X-Naver-Client-Secret` headers, `query`/`display`/`start`/`sort` params, response item fields, documented daily call limit, and error envelopes with `errorCode`/`errorMessage`; worker maps auth, rate-limit, schema, timeout, unavailable, and unknown failures to fail-closed provider errors |
| OpenAI | model/version choice for structured outputs | verified-structured-output-guarded | Official OpenAI docs confirm Responses API structured outputs through `text.format.type=json_schema` with `strict=true` and model support for current GPT-5/GPT-4.1/GPT-4o families; worker default is `gpt-5.5`, and any unverified `OPENAI_MODEL` fails closed with `openai_structured_output_model_not_verified` before an API call |
| Supabase | Free plan database limit | verified-budget-enforced | Official Supabase plan docs confirm Free project database space is 500MB; MVP keeps an internal 500MB budget and warns at `PAPER_HEALTH_DB_WARNING_BYTES=450000000` before the Free cap |
